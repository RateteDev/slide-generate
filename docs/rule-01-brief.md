# ルール: 01-brief.md

発表の前提をまとめるファイル。このファイルが軸になり、02〜04の方向性が決まる。

## 構成

```markdown
## 基本情報

| 項目 | 内容 |
|------|------|
| イベント | {イベント名} |
| 日時 | {YYYY-MM-DD} |
| 形式 | {LT / セッション / ワークショップ} |
| 持ち時間 | {例: 5分} |

## 想定する聞き手

## 一番伝えたいこと

## 発表先の空気感

## 制約・注意
```

## 各項目のルール

一番伝えたいこと：
- 必ず1文で書く。迷ったときにここへ立ち返る基準になる。

発表先の空気感
- イベントの雰囲気、好まれる発表スタイル、過去の傾向を書く。

制約・注意
- 話してはいけないこと、避けたい表現、機材制約などを書く。
- なければ「特になし」と明記する。
