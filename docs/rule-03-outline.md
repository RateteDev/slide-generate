# ルール: 03-outline.md

スライドの構成を決めるファイル。
`02-raw-notes.md` の素材を選び、順序と深さを整理する。

## 構成

```markdown
# タイトル

## 全体の方針

- 時間配分: ...
- 表現スタイル: ...
- トピック順序: ...

---

## スライド一覧

### 1. タイトル
- 見出し: ...
- 副題: ...

### 2. 自己紹介
- ...

### {N}. {トピック名} — {スライドの役割}
- 見出し: ...
- 画像: assets/xxx.png
- テキスト: ...
```

## 各スライドに書くこと

| 項目 | 内容 |
|------|------|
| 見出し | スライドの h1/h2 になるテキスト |
| 画像 | 使用する assets のファイル名 |
| テキスト | スライドに載せる文（箇条書き可） |
| 困っていたこと / 楽になったこと | chapter-title 導入スライドの内容 |

## トピックの展開パターン

各トピックは以下の順で構成する（`docs/writing-guide.md` のクラス設計に対応）。

```
{N}-1. タイトル（chapter-title）
{N}-2. 課題・背景（説明スライド）
{N}-3. Before 画像（image-only）
{N}-4. 解決策（説明スライド）
{N}-5. After 画像（image-only）
```

Before/After がない場合は省略してよい。

## 量の目安

1トピックあたり3〜7スライド。全体で30〜50行程度。
