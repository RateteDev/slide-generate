# 執筆ガイド

## 全体構成の例

1. タイトル
2. 自己紹介
3. このスライドについて
4. 章扉
5. 本編
6. まとめ
7. 参照ソース
8. フォローアップ
9. Q&A例
10. 最後のスライド

章扉と本編は必要に応じて繰り返す。

## Frontmatter

`footer` に公開URLを入れる。形式は `https://ratete.dev/works/{workSlug}`。
`workSlug` は英小文字の kebab-case で、タイトルから識別しやすい短い語に要約する。

例: タイトルが「AIをいい感じに使う設定や環境のアレコレ」→ `ai-dev-setup`

## スライドのクラス設計

### クラス一覧

| クラス | 用途 |
|--------|------|
| `title` | 発表タイトルスライド |
| `chapter-title` | セクション・トピックの導入スライド |
| `image-only` | 画像フルサイズ表示専用スライド |
| `closing-qr` | 最後のスライド（QRコード） |

### トピック導入（`chapter-title`）

セクションおよび各トピックの入り口に使う。`chapter-title` を連続させない。

```markdown
<!-- header: トピックタイトル -->
<!-- _class: chapter-title -->

# トピックタイトル

サブタイトル
```

**ルール：`chapter-title` の直後に `chapter-title` を続けない。** 間に通常スライドを挟む。

### 画像スライド（`image-only`）

画像1枚をフルサイズで表示するスライド。header / footer / ページ番号は自動で非表示。

```markdown
<!-- _class: image-only -->

![bg](assets/ファイル名.png)
```

**ルール：画像スライドにテキストを置かない。** ページ番号以外の要素はすべて除去する。

### スライドの順序ルール

各トピックは必ず **chapter-title → 説明 → 画像** の順で構成する。

```
<!-- _class: chapter-title -->  ← トピック導入
# タイトル

---

# 説明スライド               ← 文脈・背景・ポイント

---

<!-- _class: image-only -->    ← 画像（説明のあと）
![bg](assets/...)
```

いきなり画像を見せない。説明で文脈を作ってから画像で補強する。

## Header のルール

- Frontmatter の `header` はデフォルト値（発表タイトルなど）を設定する
- チャプターごとに `<!-- header: テキスト -->` ディレクティブで上書きする
- **header のテキストは、そのチャプターの `chapter-title` の h1 と完全一致させる**

```markdown
<!-- header: トピックタイトル -->
<!-- _class: chapter-title -->

# トピックタイトル
```

## スライドごとの注意

### 参照ソース

使った一次情報や公式ドキュメントを並べる。3〜6件が目安。多すぎると読まれない。

### フォローアップ

「今日の話は固定ではない」という注意書き。変化が速いテーマと相性がよい。

### Q&A例

簡単・普通・やや難の3段構成で用意しておくと、質問が出なかった場合に使える。

### 最後のスライド

`_class: closing-qr` を指定する。Header / Footer / ページ番号は自動で非表示になる。

- h1: Thank you!
- h2: 発表タイトル
- QRコード（公開URL）
- 導線テキスト＋URL

公開URLは `footer` と同じものを使う。
