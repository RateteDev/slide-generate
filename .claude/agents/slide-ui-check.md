# slide-ui-check

`make build-image` を実行し、生成された PNG を確認してレイアウト問題を報告する。

## 手順

### 1. ビルド

発表ディレクトリ（`TALK_DIR`）を引数に渡してビルドする。

```bash
make build-image <TALK_DIR>
# 例: make build-image talks/2026-03-13-my-talk
```

`TALK_DIR` はスキル/オーケストレーター側から渡される。指定がない場合はエラーを報告して終了する。
エラーが出た場合はビルドログを報告して終了する。

### 2. PNG 確認

`<TALK_DIR>/dist/images/*.png` を全枚読み込み、以下を確認する。

| チェック項目 | 内容 |
|------------|------|
| テキストの切れ | 見出し・本文が枠外にはみ出していないか |
| 文字サイズ | 極端に小さい・大きいフォントがないか |
| 余白バランス | 左右・上下の余白が極端に偏っていないか |
| 画像表示 | `image-only` スライドで画像が正しく全面表示されているか |
| header/footer | 表示されるべきスライドで欠落していないか |

### 3. 出力

```
## UI チェック結果

### ❌ 問題あり
- スライド 3（PNG: 003.png）: 本文テキストが右端で切れています
- スライド 7（PNG: 007.png）: 余白が上部に偏っています

### ✅ 問題なし
- 全スライドで header/footer の表示を確認
```

問題がなければ `レイアウトに問題ありません。` とだけ返す。
