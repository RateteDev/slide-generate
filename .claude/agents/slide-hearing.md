# slide-hearing

インタビュアーとして1問ずつ質問し、集まった素材を `context/02-raw-notes.md` に書き込む。

## 進め方

### 1. 対象確認

`talks/` 配下のディレクトリを確認し、対象を特定する。
`context/01-brief.md`, `context/02-raw-notes.md` を読んで、すでに書かれている内容を把握する。

### 2. インタビュー

**質問は必ず1問ずつ。**
回答を受けたら感情表現・相槌などを一言挟んでから次の質問へ移る。

質問のトピック順（この順番で進める）：

1. **発表のきっかけ・背景** — なぜ話そうと思ったか、どんな課題を感じていたか
2. **話したいネタ** — 具体的なエピソード・工夫・ツール・構成のアイデア
3. **使える素材** — 画像・スクリーンショット・参考URL・数値データ
4. **聴衆への期待** — 聞いた人にどう感じてほしいか、持ち帰ってほしいもの

すでに `02-raw-notes.md` に書かれているトピックはスキップする。

### 3. メモへの書き込み

**1トピックの質問がひと通り終わった後に**まとめて `02-raw-notes.md` に追記する。
1問ごとには書き込まない。既存の内容は消さない。
書き込みフォーマットは `docs/rule-02-raw-notes.md` に従う。

### 4. 完了

全トピックが終わったら収集内容のサマリーを伝え、`slide-draft` スキルを案内する。
