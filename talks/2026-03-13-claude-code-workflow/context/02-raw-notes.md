# 調査・ヒアリングメモ

## 発表のきっかけ・背景

- 個人開発で週末しか時間が取れない。毎週「どこまでやったっけ？」から再開するのが辛かった
- Claude Code を使いはじめた最初の2週間は「ただのコード補完」として使っていた
- CLAUDE.md を書いてコンテキストを渡すようにしたら、再開コストが激減した
- 「AIにプロジェクトのことを全部知ってもらう」という発想の転換が大きかった
- 同じ悩みを持つ個人開発者が多そうで、共有したいと思った

## 話したいネタ

- CLAUDE.md の書き方：プロジェクト概要・技術スタック・命名規則・禁止事項を書く
- 週次サイクルの話：土曜に30分かけてCLAUDE.mdを更新するだけで、日曜の作業が倍速になった
- コンテキストファイル設計：`context/` ディレクトリに状態ファイルを置く運用
  - `context/current-task.md` — 今やっていることと次のアクション
  - `context/decisions.md` — 設計の意思決定ログ
  - `context/gotchas.md` — ハマりポイントまとめ
- 「AIとペアプロする」ではなく「AIにプロジェクト背景を渡す」という概念の説明
- 実際にCLAUDE.mdがないときとあるときの比較エピソード：
  - ないとき：「このファイルは何のためにあるの？」から聞かれる
  - あるとき：最初からプロジェクトのルールを踏まえた提案が来る

## 使える素材

- CLAUDE.md の実例（個人ブログプロジェクトのやつ）
- context/ ディレクトリの構成スクリーンショット
- 「コンテキスト渡す前 vs 渡した後」のやりとり比較テキスト

## 聴衆への期待

- 聞いた直後に自分のプロジェクトに CLAUDE.md を作ってみようと思ってほしい
- 「AIが使えない」という思い込みを壊したい
- 笑って共感してもらいながら、帰りに試したくなるハードルの低い話にしたい
