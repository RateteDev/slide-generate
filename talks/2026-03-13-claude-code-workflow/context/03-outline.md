# Claude Code で個人開発が変わった話

## 全体の方針

- 時間配分: タイトル+自己紹介 30秒 / 課題提示 45秒 / 解決策（CLAUDE.md） 2分 / context/設計 1分 / まとめ 45秒
- 表現スタイル: 共感ファースト。「あるある」から入り、解決策を提示する。コードは最小限
- トピック順序: 課題（週次再開コスト）→ 解決（CLAUDE.md）→ 発展（context/設計）→ まとめ

---

## スライド一覧

### 1. タイトル
- 見出し: Claude Code で個人開発が変わった話
- 副題: コンテキストを渡す設計だけで爆速になる

### 2. 自己紹介
- 見出し: 自己紹介
- テキスト:
  - 個人開発歴5年 / 週末エンジニア
  - 個人ブログ・タスク管理ツールを細々と運用中
  - 最近の口癖:「CLAUDE.md に書いといて」

### 3. 課題 — chapter-title
- 見出し: 1. 毎週「どこまでやったっけ？」問題
- サブタイトル: 週末開発者の永遠の悩み

### 4. 課題の説明
- 見出し: 週末エンジニアあるある
- テキスト:
  - 土曜AM: コーヒーを淹れてやる気満々
  - 土曜AM+10分: 「あれ、このファイル何のためにあるの？」
  - 土曜PM: ようやく再開できた……もう夕方
  - → 実装時間よりコンテキスト回復時間のほうが長い

### 5. 解決策 — chapter-title
- 見出し: 2. CLAUDE.md が全部解決する
- サブタイトル: AIにプロジェクトを「知って」もらう

### 6. CLAUDE.md とは
- 見出し: CLAUDE.md = AIへの引き継ぎ書
- テキスト:
  - プロジェクトの目的・技術スタック
  - 命名規則・ディレクトリ構成
  - 「やってはいけないこと」
  - → Claude Code が起動するたびに自動で読む

### 7. Before / After
- 見出し: CLAUDE.md がないと…
- テキスト:
  - AI: 「このファイルは何のためにありますか？」
  - AI: 「どの技術スタックを使っていますか？」
  - AI: 「テストは書きますか？」
  - → 毎回同じ質問に答えている

### 8. After 説明
- 見出し: CLAUDE.md があると…
- テキスト:
  - AI が最初からルールを知っている
  - 「このプロジェクトは〇〇なので…」と提案してくれる
  - コンテキスト回復コスト = ほぼゼロ

### 9. context/ 設計 — chapter-title
- 見出し: 3. context/ で状態を永続化
- サブタイトル: 「今何をしているか」をファイルに残す

### 10. context/ 構成
- 見出し: 3ファイルで十分
- テキスト:
  - `current-task.md` — 今やっていることと次のアクション
  - `decisions.md` — 設計の意思決定ログ
  - `gotchas.md` — ハマりポイントまとめ
  - → Claude Code に毎回読ませる

### 11. まとめ
- 見出し: まとめ
- テキスト:
  - CLAUDE.md を書く → AIがプロジェクトを知る
  - context/ を育てる → 状態がAIと共有される
  - 週末30分の投資で作業時間が倍になった
  - **今夜、CLAUDE.md を作ってみてください**
